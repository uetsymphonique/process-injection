# CWLHerpaderping — Flow

**Entry:** `wmain()` → `GetPayloadBuffer()` → `Herpaderping()`  ·  **Artifact summary:** Process herpaderping loader — maps PE payload into ghost process via `NtCreateUserProcess` (suspended thread), overwrites on-disk image with IIS log decoy before resume; PPID spoofed to svchost.exe Session 0; indirect syscall + stack spoof on creation call

> **README vs code discrepancy:** README describes the classic 3-syscall pattern (`NtCreateSection` → `NtCreateProcessEx` → `NtCreateThreadEx`). Actual code uses `NtCreateUserProcess` with `THREAD_CREATE_FLAGS_CREATE_SUSPENDED` + `ResumeThread`. Only ONE indirect syscall stub is built (`NtCreateUserProcess`). Rows below reflect the code.
